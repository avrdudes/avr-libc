31/** \defgroup avr_version <avr/version.h>: AVR-LibC version macros
32 \code #include <avr/version.h> \endcode
33
34 This header file defines macros that contain version numbers and
35 strings describing the current version of AVR-LibC.
36
37 The version number itself basically consists of three pieces that
38 are separated by a dot: the major number, the minor number, and
39 the revision number. For development versions (which use an odd
40 minor number), the string representation additionally gets the
41 date code (YYYYMMDD) appended.
42
43 This file will also be included by \c <avr/io.h>. That way,
44 portable tests can be implemented using \c <avr/io.h> that can be
45 used in code that wants to remain backwards-compatible to library
46 versions prior to the date when the library version API had been
47 added, as referenced but undefined C preprocessor macros
48 automatically evaluate to 0.
49*/
50

58/** \ingroup avr_version

64 numerical value that can easily be used in numerical checks.
